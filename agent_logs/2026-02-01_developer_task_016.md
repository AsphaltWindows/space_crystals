**Methods**:
- `at_recruit_cap()`: Check if at maximum recruits
- `available_recruit_slots()`: Calculate max - current

**Recruit System**:
- Recruits are automatically generated by Recruitment Centers
- Recruitment rate based on recruitable tile area
- Each Recruitment Center has local recruit limit
- Recruits cost nothing but are limited by capacity

### Example Recruit Management
```rust
// Starting state
recruits = 5, max_recruits = 20 → available = 15 slots

// After building a building (3 recruits consumed)
recruits = 2, max_recruits = 20 → available = 18 slots

// After recruitment center generates recruits
recruits = 10, max_recruits = 20 → available = 10 slots

// At capacity
recruits = 20, max_recruits = 20 → at_recruit_cap() = true
```

## Faction Unique Mechanics (Summary)

### Global Defense Ordinance
- **Economy**: Traditional RTS resource gathering
- **Building**: Deployment Center build system
- **Special**: Power grid system affects building efficiency
- **Army**: No population cap (power is the limit)

### The Syndicate
- **Economy**: Agent workers, fast crystal gathering
- **Building**: Underground tunnel network
- **Special**: Tunnels for unit transport, invisible buildings
- **Army**: Tunnel Space limits army size

### The Cults
- **Economy**: Free auto-generated Recruits
- **Building**: Recruits enter buildings to construct them
- **Special**: Recruits consumed by buildings
- **Army**: Recruit Cap limits available workers

### Colonists
- **Economy**: Complex refinement system
- **Building**: Beacon placement, building warp-in
- **Special**: Resource refinement chains
- **Army**: Beacon Capacity limits army size
